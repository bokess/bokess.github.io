javascript组合模式

发表于 2016-09-13   |  

上一篇文章JavaScript组合模式前言介绍了为什么要用组合模式.

组合模式使用场景

组合模式的使用场景：

1 存在一批组织成某种层次体系的对象
2 希望对这批对象或其中的一部分对象实施一个操作。
组合模式的特点：
1 组合模式中只有两种类型对象：组合对象、叶子对象
2 这两种类型都实现同一批接口
3 一般我们会在组合对象中调用其方法并隐式调用"下级对象"的方法（这里我们一般采用递归的形式去做）

我们用组合模式先实现JavaScript组合模式前言的例子.

定义组合对象跟叶子对象

按照组合模式的特点，我们先定义好组合对象和叶子对象。组合对象就是我们上个例子提到的公司、分公司、部门等等，这些都属于组合对象公司，他们都有共同的特点就都由一个或多个对象组合而成。公司有分公司或在部门组合成，分公司有部门或者员工组合成。员工就是叶子对象，因为员工就是一个员工，不会出现xx和xx组合成员工。

实现组合对象定义

我先定义两个接口：

组合对象的接口:comInterface
var comInterface = new Interface("comInterface",['addChild','getChild','removeChild']);

叶子对象的接口:leafInterface
var leafInterface = new Interface('leafInterface',['hardWroking','sleeping']);

定义组合对象
function combination(name){
        //组合对象的名字
        this.name = name;
        //定义一个type属性标明它是组合对象方便后面判断
        this.type = 'combination';
        //组合对象下面有分公司或者部门或者员工,所以要定义个数组存放他们的子类
        this.childs = [];
    };
    //实现组合对象addChild(添加子类)方法
    combination.prototype.addChild = function(name){
        //把传进来的子类推进childs数组
        this.childs.push(name);
        //还是为了链式操作
        return this;
    };
    //实现组合对象getChild(获取子类)方法
    combination.prototype.getChild = function(name){
        //定义一个数组存放找到的叶子对象
        var childsArr = [];
        if(name && this.name !== name){
            //如果传了名字并且名字不是总公司名字
            //程序执行下面代码
            this.childs.forEach(function(item){
                //传入名字是总公司直接子类的情况
                if(item.name === name && item.type === 'combination'){
                    item.childs.forEach(function(item){
                        //判断item是否是组合对象
                        if(item.type === 'combination'){
                            //如果是组合对象肯定有childs的子类
                            //我们通过递归arguments.callee(指该函数本身)再遍历item的子类
                            item.childs.forEach(arguments.callee);
                        }else if(item.type === 'leaf'){
                            //直到遍历item为叶子对象的时候把item推入数组
                            //遍历完成就会得到总公司直接子类的所有叶子对象
                            childsArr.push(item);
                        };
                    });
                };
                //传入名字是2级3期.....n级子类的情况
                if(item.name !== name && item.type === 'combination'){
                    //递归从新遍历下一级
                    item.childs.forEach(arguments.callee);
                };
                //传入名字是叶子对象的情况
                if(item.name === name && item.type === 'leaf'){
                    childsArr.push(item);
                };
            });
        }else{
            //如果不传名字或者传总公司名字就遍历整个公司
            //this.childs指总公司下的分公司或者部门
            this.childs.forEach(function(item){
                //判断item是否是组合对象
                if(item.type === 'combination'){
                    //如果是组合对象肯定有childs的子类
                    //我们通过递归arguments.callee(指该函数本身)再遍历item的子类
                    item.childs.forEach(arguments.callee);
                }else if(item.type === 'leaf'){
                    //直到遍历item为叶子对象的时候把item推入数组
                    //遍历完成就会得到总公司的所有叶子对象
                    childsArr.push(item);
                };
            });
        };
        //把获取到的叶子对象返回去
        return childsArr;
    };
    //现实组合对象removeChild(移除子类)方法
    combination.prototype.removeChild = function(name){
        var index = this.childs.indexOf(name);
        if(index>-1){
            this.childs.splice(index,1);
        }else{
            console.warn("没找到"+name);
        };
        return this;
    };
    //现象叶子对象hardWroking(努力工作)方法
    combination.prototype.hardWroking = function(name){
        //努力工作方法最终实现的是叶子对象
        //所以要通过getChild获取的叶子对象
        var arr = this.getChild(name);
        //遍历获取到的所有叶子对象
        for(var i =0;i<arr.length;i++){
            //让所有叶子对象执行hardWroking
            arr[i].hardWroking();
        };
    };
    //现象叶子对象sleeping(努力睡觉...)方法
    //原来跟hardWroking一样,不做多讲解
    combination.prototype.sleeping = function(name){
        var arr = this.getChild(name);
        for(var i=0;i<arr.length;i++){
            arr[i].sleeping();
        };
    };

实现叶子对象定义

定义叶子对象
function leaf(name){
        //叶子对象的名字
        this.name = name;
        //定义type属性标明是叶子对象方便判断
        this.type = 'leaf';
    };
    //实现组合对象addChild方法
    leaf.prototype.addChild = function(){
        //因为是叶子对象,他下面不会再有子类了,所以不存在添加子类这里可以是个空函数,
        或者调用叶子对象这个方法的时候可以抛个错误
    };
    //实现组合对象removeChild方法
    leaf.prototype.removeChild = function(){
        //既然叶子对象不存在添加子类方法,相应也不存在移除子类方法,所以这里可以是个空函数,
        或者调用叶子对象这个方法的时候可以抛个错误
    };
    //实现组合对象getChild方法
    leaf.prototype.getChild = function(){
        //叶子对象不存在子类,所以也应该没有获取子类的说法.所以这里可以是个空函数,
        或者调用叶子对象这个方法的时候可以抛个错误
    };
    //实现叶子对象hardWroking方法
    leaf.prototype.hardWroking = function(){
        document.write(this.name+" 努力工作...<br/>");
    };
    //实现叶子对象sleeping方法
    leaf.prototype.sleeping = function(){
        document.write(this.name+"努力休息迎接下次加班...<br/>");
    };

    这里有人会问你叶子对象都没用到组合对象的方法,为什么还要实现组合对象的方法呢?
    我这里只是按照组合对象的特点:2这两种类型都实现同一批接口.所以我把两个接口的方法,在
    组合对象和叶子对象都实现了.

创建公司

用叶子对象实例员工

var xLi = new leaf("小李");
var xZeng = new leaf("小曾");
var xZhang = new leaf("小张");
var xLiu = new leaf("小刘");
var xMo = new leaf("小莫");
var xLin = new leaf("小林");
var xZhuang = new leaf("小庄");
var xJiang = new leaf("小江");
var xMing = new leaf("小明");

用组合对象实例部门

var development = new combination("开发部门");
//小李,小曾,小张,小刘,小莫,小林都是入职到开发部门
development.addChilds(xLi).addChilds(xZeng).addChilds(xZhang).addChilds(xLiu).addChilds(xMo).addChilds(xLin);

var finance = new combination("财务部门");
//小庄,小江,小明是入职到财务部门
finance.addPerson(xZhuang).addPerson(xJiang).addPerson(xMing);

用组合对象实例总公司

var company = new combination("总公司");
//公司下面有开发部门(development)和财务部门(finance)
company.addChilds(development).addChilds(finance);

测试

好了现在公司构建完了,我们可以用company.hardWroking()或者company.hardWroking(‘总公司’),让全公司的员工努力工作了.company.hardWroking(‘财务部’)让财务部全体员工努力工作.company.hardWroking(‘小林’)让小林努力工作.

改需求

上个列子基本都实现了,好那我们来看看改需求,公司挣钱了,开分公司了.需求要实现传分公司名字,让分公司全体努力工作.这时候应该怎么改?
其他代码什么不用变,只要把创建公司的代码改就可以.
我们来看例子:

//实例员工
var xHong = new leaf('小红');
var xMing = new leaf('小明');
var xDong = new leaf('小东');
var xJiang = new leaf('小江');
var xLi = new leaf("小李");
var xZeng = new leaf("小曾");
var xZhang = new leaf("小张");
var xLiu = new leaf("小刘");
var xMo = new leaf("小莫");
var xLin = new leaf("小林");
var xZhuang = new leaf("小庄");

//实例广州分公司的部门
var GZfinance = new combination('广州财务部门');
GZfinance.addChild(xHong);

var GZdevelop = new combination('广州开发部门');
GZdevelop.addChild(xMing).addChild(xDong).addChild(xJiang);

//实例北京分公司的部门
var BJfinance = new combination('北京财务部门');
BJfinance.addChild(xLi).addChild(xZeng);

var BJdevelop = new combination('北京开发部门');
BJdevelop.addChild(xZhang).addChild(xLiu).addChild(xMo).addChild(xLin).addChild(xZhuang);

//实例广州分公司
var GZfiliale = new combination('广州分公司');
GZfiliale.addChild(GZfinance).addChild(GZdevelop);

//实例北京分公司
var BJfiliale = new combination('北京分公司');
BJfiliale.addChild(BJfinance).addChild(BJdevelop);

//实例总部
var headOffice = new combination('总部');
headOffice.addChild(GZfiliale).addChild(BJfiliale);

//调用方法还是一样
headOffice.hardWroking();//全公司全体努力工作

headOffice.hardWroking('广州分公司')//广州分公司全体努力工作

headOffice.hardWroking('广州财务部')//广州财务部全体努力工作

headOffice.hardWroking('小林')//小林努力工作

headOffice.sleeping()//全公司全体努力工作

我们看到只需要改变公司主体就行了,不会上个例子(JavaScript组合模式前言)要添加分公司类,函数逻辑也要改.对比之下,是不是觉得组合模式更能是适应需求变动.这就是组合模式,所有的实例都是通过组合对象和叶子对象,实例化出来的.
